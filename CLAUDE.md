# SMR Centr — правила проекта

## Что это за проект
SMR Centr — ERP-платформа для управления строительными объектами, финансами, снабжением, производством, ПТО и связанными бизнес-процессами.

## Технологический стек
- PHP 8.4
- PostgreSQL
- nginx
- php-fpm
- Redis
- нативный JavaScript без фреймворков и библиотек

## Жесткие ограничения
- если есть авторизация, сессии хранить в Redis
- не использовать frontend frameworks
- не использовать UI libraries без отдельного согласования
- все секреты хранить вне кода
- конфигурацию вести через .env

## Стандарты
- SOLID
- YAGNI
- DRY
- KISS
- PSR-12
- PSR-4
- PHPDOC для сложной логики
- UNIT tests для критической функциональности

## Архитектурные правила
- начинать как modular monolith
- тонкие controllers
- бизнес-логика в services/actions
- доступ к данным через repositories
- обязательная валидация входных данных
- явные migrations
- без скрытой магии
- без ненужных рефакторингов

## Роль Claude
Claude:
- помогает декомпозировать задачи
- пишет технические задания
- проверяет архитектуру
- делает review
- следит за соблюдением стека

## Что Claude не должен делать
- менять стек без прямой команды
- тащить фреймворки во frontend
- одобрять массовые несвязанные изменения
- разрешать архитектурный хаос
